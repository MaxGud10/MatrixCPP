# Matrix
Проект создан в рамках курса по
C++ Константина Владимирова. Задание состояло в том, чтобы реализовать класс, представляющий матрицу и предоставляющий метод для вычисления определителя.

# 2 Уровень

Второй уровень посвящен перемножению матриц. Оказывается, для
достаточно длинных цепочек матриц, порядок их перемножения, будучи
выбран правильно, позволяет сэкономить массу машинного времени.

Предположим у вас есть матрица A размера 10 × 30, B размера 30 × 5 и
C размера 5 × 60.

Вычисление их произведения как A(BC) займёт (30×5×60) + (10×30×60) =
27000 операций умножения. В то же время, их произведение как (AB)C
даст тот же результат за 4500 операций.



## Входные данные: пары размеров матриц

***Например***
```
5
30 35 15 5 10
```

Здесь задана цепочка из четырёх матриц размерами 30 x 35, 35 x 15, 15
x 5 и 5 x 10.

Для них возможны три операции умножения и разные расстановки скобок:

(((A * B) * C) * D) это порядок 0 1 2
((A * B) * (C * D)) это порядок 0 2 1 и он же 2 0 1

И т.д.

## Выход:
- оптимальный порядок
- множитель насколько он лучше по
количеству умножений, чем порядок 0 1 2.
Если такой порядок более чем один, выводим лексикографически
наименьший, например 0 2 1 а не 2 0 1.


## Сборка

### Режим **verbose**


Для более наглядного объяснения результата доступен режим:
```
./build/matrix_chain --verbose
```
<details>
В этом режиме программа печатает:
- `optimal_order` — оптимальный порядок операций
- `naive_order` — наивный порядок (0 1 2 ...)
- `naive_cost` — число скалярных умножений в наивном порядке
- `optimal_cost` — число скалярных умножений в оптимальном порядке
- `factor` — отношение naive_cost / optimal_cost

Пимер:
```
5
30 35 15 5 10
optimal_order: 1 0 2
naive_order: 0 1 2
naive_cost: 19500
optimal_cost: 9375
factor: 2.080000
```

</details>
